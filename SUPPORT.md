# Support

## Questions and Ideas

Use
[GitHub Discussions](https://github.com/omobolajiadeyan/phishguard-ai/discussions)
for setup questions, design ideas, integration examples, and general
conversation.

## Bugs and Detection Accuracy

Open a public issue for reproducible bugs, false positives, false negatives,
and documentation problems. Use synthetic or public-safe samples and remove
credentials, private email, personal data, and active malicious payloads.

## Security Vulnerabilities

Do not report exploitable vulnerabilities in a public issue. Follow
[SECURITY.md](SECURITY.md) and use GitHub private vulnerability reporting.

## Response Expectations

The project is maintained on a best-effort basis. Reports with a minimal
reproduction, Python version, operating system, expected result, and actual
result are the easiest to investigate promptly.
